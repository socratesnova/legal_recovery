CONTEXTO GENERAL:
Construir Legal Recovery OS, una plataforma para oficinas legales que gestionan cartera castigada bancaria de bancos, fondos, financieras, cooperativas y entidades cedentes. El sistema debe priorizar recuperacion neta, trazabilidad, gobierno legal de datos, auditoria y cumplimiento. No construir funcionalidades para scraping, acceso no autorizado, contacto a terceros, uso indebido de JCE/TSS/buros ni WhatsApp para cobro directo masivo.

REGLAS CRITICAS:
1. Todo dato debe tener fuente, fecha, permiso de uso, restriccion, confiabilidad y auditoria.
2. Bloquear datos sin fuente legal o contractual.
3. Bloquear contacto a terceros no autorizados.
4. WhatsApp solo debe ser canal neutral/restringido con opt-in y reglas de politica.
5. TSS/JCE/buros solo mediante integraciones autorizadas, convenio, consentimiento, orden o base legal valida.
6. Las decisiones legales sensibles requieren revision humana.
7. Mantener logs inmutables de acciones criticas.
8. Implementar RBAC/ABAC por rol, institucion, cartera y sensibilidad del dato.

OBJETIVO:
Construir API backend para Legal Recovery OS.

MODULOS API:
/auth, /users, /institutions, /portfolios, /cases, /documents, /data-passports, /contacts, /consents, /scores, /agreements, /payments, /disputes, /communications, /reports, /audit.

REQUERIMIENTOS:
- Validacion fuerte de input.
- RBAC/ABAC.
- Auditoria en cada create/update/delete/download/contact.
- Bloqueo de datos o contactos no permitidos.
- No enviar mensajes si data_passport no permite contacto.
- No mostrar datos restringidos a roles no autorizados.
- API preparada para colas y tareas asincronas.

ENTREGABLE:
Crea routers, modelos, servicios, DTO/schemas, middlewares de auditoria y pruebas unitarias.
